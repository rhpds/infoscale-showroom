= Multi-Site Disaster Recovery for Healthcare Systems with InfoScale & OpenShift Virtualization
include::partial$_attributes.adoc[]

Welcome to this hands-on workshop! You will configure and validate a multi-site disaster recovery solution using InfoScale for Kubernetes {infoscale_version} and OpenShift Virtualization {ocpv_version}.

== What you'll learn

In this workshop, you will:

* Verify and validate InfoScale for Kubernetes across multiple OpenShift clusters
* Install and configure the InfoScale DR Manager on both clusters
* Deploy a healthcare VM application on OpenShift Virtualization
* Configure asynchronous data replication using InfoScale custom resources
* Create and manage a DisasterRecoveryPlan using Kubernetes-native tooling
* Execute and validate a complete workload failover between clusters

== Who this is for

This workshop is designed for Platform Engineers, Storage Administrators, SREs, and OpenShift Virtualization Architects with intermediate experience who need to implement enterprise-grade disaster recovery for healthcare workloads.

**Experience level**: Intermediate

== Prerequisites

Before starting this workshop, you should have:

* Familiarity with OpenShift and the `oc` CLI
* Basic understanding of Kubernetes custom resources (CRDs)
* Understanding of storage concepts (PersistentVolumes, StorageClasses, replication)
* Cluster-admin access to both `{primary_cluster}` and `{secondary_cluster}`

== Workshop environment

This workshop uses two pre-configured OpenShift {ocp_version} SNO clusters representing a primary production site and a geographically separate disaster recovery site.

You will have access to:
